Deep dive — missing income14:00 – 19:30
Client

~70% of retail customers have no income in gold. Marketing and credit both complain. Your approach?

Will Tran
  1. Measure — fill rate overall, by channel/product/region; CRM vs core vs gold (dq_income_completeness.sql).
  2. Segment root cause — optional UI (~45%), CRM not in ETL (~20%), mapping bugs (~5%), refused/N/A, migration gap.
  3. Source fix — PO policy: mortgage above limit X requires income; block submit in app for requires_income products.
  4. ETL fix — add CRM income to silver join; quarantine conflicts; remove legacy NVL(income,0) anti-pattern.
  5. DQ gates — CRITICAL if mortgage cohort <95% complete; block gold publish (dq_contract.py).
  6. Estimates separate — estimated_* + is_imputed; never silent overwrite of declared.
  7. Governance by use case — marketing may use flagged estimates; final credit = declared or bureau-verified only.
Architecture19:30 – 23:00

Whiteboard verbally: customer 360 on AWS with Oracle still system of record.

AWS & production23:00 – 26:30
Client

Three quick ones: Glue vs Lambda vs EMR? Idempotent daily load? Last production incident?

Will Tran
  • Glue — scheduled Spark ETL, JDBC catalog, bookmarks. Lambda — small event transforms, API micro-batch. EMR — heavy custom Spark or long-running clusters when Glue limits bite.
  • Idempotent daily: partition dt= + run_id; staging folder then atomic rename; overwrite only same partition; watermark on last_updated_ts.
  • Incident (hybrid): VPN blip → half Oracle extract landed → downstream ran → BI wrong counts. Fix: _SUCCESS per extract, Step Functions/Airflow won’t trigger silver without it; comms template to BI before refresh; postmortem in docs/05-real-world-issues.md.
SQL / modeling26:30 – 27:30
